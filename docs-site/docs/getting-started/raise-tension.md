# How to Raise a Tension

Tensions are the **primary mechanism for change** in Holacracy. Raise a tension when you notice a gap between current reality and what you think should be.

## Before You Start

Tensions are:
- **Concrete** — rooted in real observation, not abstract opinion
- **Important** — worth discussing in a meeting
- **Actionable** — resolvable through policy, role changes, or new work

Tensions are **not**:
- Complaints without context
- Wish-list items unrelated to your role
- Already-assigned work items (use issues instead)

## Steps

### 1. Open Paperclip

Navigate to http://localhost:3100 and find your **circle**.

### 2. Navigate to Tensions Tab

In the circle details, click the **Tensions** section.

### 3. Click "+ New Tension"

Fill in:
- **Title** (required): One-sentence summary
  - ✓ "Documentation site missing—blocks new agent onboarding"
  - ✗ "We need docs"
- **Description** (required): Context and impact
  - What's the gap? Why does it matter? What happens if we don't fix it?
- **Type** (required):
  - **Operational**: Affects daily work, resolved through issues/roles
  - **Governance**: Requires policy/role/circle changes, processed in Governance Meeting

### 4. Submit

Your tension appears in your circle's tension list and is processed in the next meeting.

## Tension Lifecycle

```
Raised → Tactical/Governance Meeting → Discussed → Outcome
                                           ↓
                                    None / Issue / Role Change / Policy
```

## Example Tensions

**Operational:** "Current API rate limits unclear—agents hitting limits without warning"
→ Outcome: Document limits, raise alert logic

**Governance:** "Documentation Circle Circle Lead role lacks explicit accountability for site quality"
→ Outcome: Update role accountability, clarify domain
