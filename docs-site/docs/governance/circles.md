# Holacracy Circles

Circles are **teams organized around purpose**, not hierarchy. Multiple circles can nest, with representatives connecting them.

## Circle Structure

Every circle has these core roles:

| Role | Purpose |
|------|---------|
| **Circle Lead** | Owns circle evolution, fills vacancies, breaks ties in governance |
| **Facilitator** | Runs Tactical and Governance meetings |
| **Secretary** | Records decisions, maintains records, publishes minutes |
| **Circle Rep** | Represents circle in parent circle |

## Domain Roles

Specialized roles focused on specific work:
- **Documentation Lead** — owns docs site
- **API Developer** — owns API design
- **DevOps** — owns infrastructure
- **etc.**

Each role has a **purpose** (why), **domain** (what it owns), and **accountabilities** (responsibilities).

## Sub-circles

Circles can contain sub-circles:

```
General Company Circle
├── Operations Circle
│   ├── Documentation Sub-circle
│   └── Infrastructure Sub-circle
└── R&D Circle
```

The **Circle Rep** from each sub-circle participates in the parent circle's meetings.

## Governance Meetings

Run bi-weekly to:
- Process governance tensions
- Add/remove/modify roles
- Create new policies
- Update circle purpose or domain

## Tactical Meetings

Run weekly to:
- Process operational tensions
- Assign work to roles
- Discuss blockers
- Sync on progress
