# Governance Meetings

Governance meetings process **governance tensions** — proposed changes to roles, circles, policies. Held bi-weekly.

## Meeting Agenda (120 min)

1. **Checkin** (5 min) — Personal update
2. **Governance Processing** (100 min) — Process each tension
3. **Closing** (15 min) — Confirm decisions, publish updates

## Governance Tension Examples

- "Documentation Lead role lacks accountability for content quality"
- "Need a policy: code changes require approval"
- "Create a new Infrastructure Circle"
- "API Developer role domain excludes database schema"

## Processing Flow

For each governance tension:

1. **Proposal** — Concrete change proposed
2. **Clarifying Questions** — Fact-finding only
3. **Reactions** — Everyone shares impact
4. **Amendment Moment** — Refine based on feedback
5. **Decision** — Integration or voting

## Example Governance Meeting

**Tension:** "Documentation Lead role needs clarity on content ownership"

- **Proposal:** Update domain to include "all user-facing content"
- **Clarifying Q:** "Internal wiki docs?" → "No, just external site."
- **Reactions:** "Good ownership." / "Might conflict with Circle Lead domain."
- **Amendment:** Refine domain language
- **Decision:** Approved. Update in Paperclip.
