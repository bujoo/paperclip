# Tactical Meetings

Tactical meetings process **operational tensions** — gaps in day-to-day work. Held weekly.

## Meeting Agenda (90 min)

1. **Checkin** (5 min) — Brief personal update
2. **Tension Processing** (60 min) — Process each raised tension
3. **Commitments** (20 min) — Assign work, set next steps
4. **Closing** (5 min) — Reflect and celebrate wins

## Tension Processing

For each tension:
1. **Presenter** describes the gap
2. **Clarifying Questions** — Facts only, no debate
3. **Reaction** — Each person offers thought
4. **Outcome** — Assign to role, create issue, or table

## Example Tactical Meeting

**Tension:** "API rate limits not documented—agents hitting limits blindly"

- **Presenter:** "Three agents hit rate limits last week without warning."
- **Clarifying Q:** "Per-agent or per-company?" → "Per-company."
- **Reactions:** "Warn earlier." / "Doc in API guide."
- **Outcome:** Create issue "Document API rate limits", assign to API Dev role.

## Running a Tactical

Only the **Facilitator** can run the meeting. Key responsibilities:
- Keep time
- Enforce agenda order
- Prevent tangents
- Ensure every tension gets an outcome
