# Company Strategy v1 — "My AI Company"
**Author:** Strategist (agent aec05dae)
**Status:** Published (v1) — 2026-05-27
**Review cadence:** Monthly (last Thursday of each month)

## 1. Identity & North Star
We are an autonomous, AI-native company. Humans set direction; agents execute. Every workflow that *can* be agent-driven *should* be agent-driven, with humans focused on judgment, governance, and customer empathy.

**North Star (12-month):** Demonstrate a sustainably profitable workflow run end-to-end by AI agents, with human oversight measured in minutes-per-day, not hours.

## 2. Strategy Heuristics (rules of thumb, not goals)
These guide every prioritization decision. When two roles disagree, the heuristic with the higher number wins.

1. **Prefer AI-automatable workflows.** If a task could plausibly be done by an agent within 30 days of tooling work, prefer it over one that requires permanent human labor.
2. **Optimize for agent throughput, not headcount.** Success = more issues closed per agent-hour, not more agents hired. Add an agent only when an existing role is provably saturated.
3. **Ship a thin slice weekly.** A crap v1 in production beats a polished v3 in draft. Every circle publishes *something* externally observable each week.
4. **Compounding assets > one-off output.** Prefer building a tool, template, or doc that future runs reuse over solving the same problem twice manually.
5. **Customer evidence beats internal opinion.** A real lead, user signup, or paying customer outranks any internal debate about strategy.

## 3. OKR Framework — Q2 2026
*Two objectives, three KRs each. KRs are quantitative and measurable from Paperclip + connected systems.*

### O1: Prove the autonomous-execution loop
- KR1.1: ≥ 25 issues moved to `done` by agents (not humans) per week, sustained for 4 consecutive weeks.
- KR1.2: < 10% of agent-completed issues require human rework (measured by reopen rate).
- KR1.3: Mean human-touch-time per closed issue ≤ 5 minutes.

### O2: First external customer signal
- KR2.1: 3 ICPs documented with named example accounts (Sales circle owns).
- KR2.2: 10 outbound conversations with prospects logged in a CRM.
- KR2.3: 1 paying customer or letter of intent secured by end of Q2.

## 4. What this strategy de-prioritizes (explicit no's)
- Custom-built infra where an off-the-shelf SaaS solves it for < $200/mo.
- Brand polish before product–market fit (logo v2 waits until paid customer #5).
- Hiring more specialist agents before existing ones are saturated.

## 5. Operating cadence
- **Daily:** Each circle's Lead processes inbox.
- **Weekly:** 1 thin-slice ship per circle. Tactical meetings to triage tensions.
- **Monthly:** OKR review (last Thursday). Heuristics editable only at this review.
- **Quarterly:** Re-baseline OKRs and re-rank heuristics.

## 6. Hand-offs
- **OKR Architect:** owns *measurement* of KRs (dashboards, weekly numbers).
- **Product / Engineering Leads:** translate O1 into roadmap items (MYA-10 unblocked by this doc).
- **Sales Lead:** owns O2 fully.
- **Circle Leads:** ensure each weekly ship maps to at least one KR.
